What Are GPT Actions?

By default, custom GPTs can only chat. Actions let them interact with external services ‚Äî save data to spreadsheets, fetch information from APIs, send emails, and more.

Think of Actions as teaching your GPT new skills beyond conversation.

üìä Save Data

Store information in Airtable, Notion, Google Sheets, or any database.

üîç Fetch Info

Pull data from job boards, weather APIs, or your own services.

üìß Take Action

Send emails, create calendar events, post to Slack, and more.

Before You Start

  • Requirement met: ChatGPT Plus, Team, or Enterprise ‚Äî Custom GPTs require a paid plan ($20/month for Plus)
  • Requirement met: An external service with an API ‚Äî We'll use Airtable (free tier works great)
  • Requirement met: 15-20 minutes ‚Äî That's all it takes!

Let's Build It

We'll create a GPT that saves internship opportunities to an Airtable database ‚Äî perfect for job hunting.

1

Set Up Your Airtable Base

Go to airtable.com and create a free account. Create a new base called "Internships" with these columns:

  • Company (Single line text)
  • Role (Single line text)
  • Location (Single line text)
  • Link (URL)
  • Notes (Long text)
2

Get Your Airtable Credentials

You'll need two things from Airtable:

  1. Go to airtable.com/create/tokens and create a Personal Access Token with data.records:write scope
  2. Open your base and copy the Base ID from the URL: airtable.com/appXXXXXXXX/...
üí° Keep these safe! You'll paste these into ChatGPT in a moment.
3

Create Your Custom GPT

In ChatGPT, click your profile ‚Üí My GPTs ‚Üí Create a GPT

Give it a name like "Internship Hunter" and add instructions:

Instructions 
You help users find and track internship opportunities.

When the user shares a job posting or describes an opportunity:
1. Extract the company name, role, location, and application link
2. Save it to the Airtable database using the action
3. Confirm what was saved

You can also search the web for internships when asked.
Be encouraging and helpful ‚Äî job hunting is stressful!
4

Add the Action

Scroll down to Actions ‚Üí Create new action

What is an OpenAPI schema? Think of it as a "menu" for your API. It tells the GPT: "Here's what you can order (endpoints), here's what info you need to provide (fields), and here's where to send it (the URL)." The GPT reads this menu to know how to talk to Airtable.

Paste this schema into the action editor. You only need to change ONE thing: replace YOUR_BASE_ID with your actual Base ID from Step 2.

OpenAPI Schema (YAML) 
openapi: "3.1.0"
info:
  title: "Airtable Internships"
  version: "1.0.0"
  description: "Save internship opportunities to Airtable"
servers:
  - url: "https://api.airtable.com/v0/YOUR_BASE_ID"
paths:
  /Internships:
    post:
      operationId: "addInternship"
      summary: "Add an internship to the database"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: "object"
              properties:
                fields:
                  type: "object"
                  properties:
                    Company:
                      type: "string"
                      description: "Company name"
                    Role:
                      type: "string"
                      description: "Job title"
                    Location:
                      type: "string"
                      description: "Job location"
                    Link:
                      type: "string"
                      description: "Application URL"
                    Notes:
                      type: "string"
                      description: "Additional notes"
                  required:
                    - "Company"
                    - "Role"
      responses:
        "200":
          description: "Successfully added"

Breaking Down the Schema

Here's what each part means (you don't need to memorize this ‚Äî just understand the structure):

Header
openapi, info ‚Äî Just metadata. Tells the system this is a valid OpenAPI 3.1 file.
Server URL
servers ‚Üí url ‚Äî This is where you put your Base ID! It's the address where requests go: https://api.airtable.com/v0/appXXXXXX
Path
/Internships ‚Äî The table name. Must exactly match your Airtable table name (case-sensitive!).
Method
post ‚Äî We're creating new records. (Use get to read, patch to update, delete to remove.)
Fields
properties ‚Üí Company, Role, etc. ‚Äî These are your Airtable columns. Names must match exactly. The description helps the GPT understand what to put in each field.
Common mistake: If your Airtable table is named "Job Listings" (with a space), the path must be /Job%20Listings ‚Äî spaces become %20.
5

Add Authentication

Below the schema, click Authentication ‚Üí API Key

  • API Key: Paste your Personal Access Token
  • Auth Type: Bearer

Click Save, then Update (or Create) in the top right!

6

Test It Out!

Open your new GPT and try:

"I found an internship at Stripe for a Software Engineering Intern in San Francisco. Here's the link: stripe.com/jobs/123"

Your GPT should save it to Airtable and confirm. Check your Airtable base ‚Äî the data should appear!

Troubleshooting

Running into issues? Here are the most common problems and how to fix them.

  • Why this happens

    Your API key is missing, incorrect, or doesn't have the right permissions.

    How to fix it

    • Double-check you selected Bearer as the Auth Type (not "Basic" or "Custom")
    • Make sure you copied the entire API key ‚Äî they're long!
    • Verify your Airtable token has the data.records:write scope
    • Check that the token has access to the specific base you're using
    Quick test: Try your token in the Airtable API playground to verify it works.

  • Why this happens

    The Base ID in your schema URL or the table name in the path doesn't match what's in Airtable.

    How to fix it

    • Check your Base ID ‚Äî it starts with app (e.g., appABC123xyz)
    • The table name in your path must exactly match your Airtable table name, including capitalization
    • If your table is named "Internships", the path should be /Internships
    • Spaces in table names need to be URL-encoded: "Job List" ‚Üí /Job%20List
    Quick test: Open your Airtable base and check the URL ‚Äî it contains your Base ID right after airtable.com/

  • Why this happens

    The field names in your schema don't match your Airtable column names, or you're sending the wrong data type.

    How to fix it

    • Field names are case-sensitive: "Company" ‚â† "company"
    • Check for typos in field names
    • Make sure you're using the correct data types (strings for text, numbers for numeric fields)
    • Airtable expects data wrapped in a fields object ‚Äî the schema above handles this
    Quick test: Open Airtable and copy/paste the exact column names into your schema to avoid typos.

  • Why this happens

    Your YAML/JSON has syntax errors ‚Äî usually indentation or missing quotes.

    How to fix it

    • YAML is indentation-sensitive ‚Äî use consistent spaces (2 spaces per level)
    • Never mix tabs and spaces
    • Wrap URLs and special characters in quotes
    • Check for missing colons after property names
    Quick test: Paste your schema into a YAML validator to find syntax errors.

  • Why this happens

    The GPT doesn't know when to use the action, or the action isn't properly configured.

    How to fix it

    • Add clear instructions telling the GPT when to use the action (see Step 3)
    • Make sure the action has a descriptive summary and description
    • Try explicitly asking: "Save this to the database"
    • Check that you clicked Save after adding the action
    • Verify the action appears in the GPT's configuration panel
    Quick test: Ask the GPT "What actions do you have available?" ‚Äî it should describe the Airtable action.

  • Why this happens

    Airtable's free tier limits you to 5 requests per second. If you're saving lots of data quickly, you'll hit this limit.

    How to fix it

    • Wait a few seconds and try again
    • For bulk operations, save items one at a time with pauses between
    • Consider upgrading to Airtable's paid plan for higher limits
    Good news: For normal use (saving a few internships at a time), you won't hit this limit.

  • Why this happens

    The server URL in your schema is incorrect or the service is temporarily down.

    How to fix it

    • Verify the server URL is exactly https://api.airtable.com/v0/YOUR_BASE_ID
    • Make sure you're using https:// not http://
    • Check Airtable's status page for outages
    • Try again in a few minutes ‚Äî sometimes it's just a temporary glitch
Still stuck? Try the "Test" button in the GPT action configuration ‚Äî it shows the exact request and response, which helps pinpoint the issue. You can also check OpenAI's GPT Community Forum for help.
