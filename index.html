What are GPT Actions?

By default, custom GPTs can only have conversations. They can't save data, fetch live information, or connect to other services.

Actions change that. They let your GPT talk to external services through their APIs â€” like saving job listings to a spreadsheet, fetching weather data, or sending emails.

Think of it this way

Actions are like teaching your GPT to use apps on your behalf. You're giving it permission to press buttons in other services.

What is an OpenAPI schema?

It's a "menu" that tells the GPT what the API can do. It describes: where to send requests, what data to include, and what responses to expect. The GPT reads this menu to know how to talk to Airtable.

Paste this schema into the action editor. Replace YOUR_BASE_ID with your actual Base ID from Step 2:

OpenAPI Schema (YAML) 
openapi: "3.1.0"
info:
  title: "Airtable Internships"
  version: "1.0.0"
  description: "Save internship opportunities to Airtable"
servers:
  - url: "https://api.airtable.com/v0/YOUR_BASE_ID"
paths:
  /Internships:
    post:
      operationId: "addInternship"
      summary: "Add an internship to the database"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: "object"
              properties:
                fields:
                  type: "object"
                  properties:
                    Company:
                      type: "string"
                      description: "Company name"
                    Role:
                      type: "string"
                      description: "Job title"
                    Location:
                      type: "string"
                      description: "Job location"
                    Link:
                      type: "string"
                      description: "Application URL"
                    Notes:
                      type: "string"
                      description: "Additional notes"
                  required:
                    - "Company"
                    - "Role"
      responses:
        "200":
          description: "Successfully added"

Understanding the schema

Here's what each part does:

Part What it means
servers â†’ url This is where you put your Base ID. It's the address where requests get sent.
/Internships Your table name. Must match exactly, including capitalization.
post We're creating new records. Use get to read, patch to update.
properties Your Airtable columns. Names must match exactly.

Common mistake: If your table name has a space (like "Job Listings"), encode it as /Job%20Listings

Troubleshooting

Running into errors? Here are the most common issues and how to fix them.

  • Why this happens

    Your API key is missing, incorrect, or doesn't have the right permissions.

    How to fix it

    • Make sure you selected Bearer as the Auth Type
    • Check that you copied the entire API key â€” they're long!
    • Verify your token has the data.records:write permission

  • Why this happens

    The Base ID or table name in your schema doesn't match what's in Airtable.

    How to fix it

    • Check your Base ID â€” it should start with app
    • Make sure your table name matches exactly (case-sensitive)
    • Encode spaces as %20

  • Why this happens

    Field names in your schema don't match your Airtable column names.

    How to fix it

    • Field names are case-sensitive: "Company" â‰  "company"
    • Copy column names directly from Airtable to avoid typos

  • Why this happens

    Your YAML has syntax errors â€” usually indentation or missing quotes.

    How to fix it

    • YAML uses spaces for indentation (2 spaces per level), not tabs
    • Wrap URLs and special characters in quotes
    • Use a YAML validator to check for errors

  • Why this happens

    The GPT doesn't know when to use the action.

    How to fix it

    • Add clear instructions telling the GPT when to use the action
    • Try explicitly saying "Save this to the database"
    • Ask the GPT "What actions do you have?" â€” it should list Airtable
